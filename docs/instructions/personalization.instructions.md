# Personalization rules

- Deliver vocabulary steering before speaker profiles, and speaker profiles before adapter training.
- Solve rare terms with contextual biasing before acoustic fine-tuning.
- Keep vocabulary entry/set schemas in `@speech/protocol` and structural validation/revision extraction in `@speech/context-bias`.
- Apply vocabulary revisions and profile swaps only at utterance boundaries.
- Refuse profiles whose base model identity, graph-contract hash, checksum, or regression gate fails.
- Preserve one-click fallback to the generic model.
- Use `projected` internally for loud/clear enrollment and warn users not to strain or scream.
- Enrollment calibration should request browser processing off where supported, inspect actual track settings, keep room-noise and normal-baseline guidance relative to the user's own RMS, and store only derived metrics unless the user explicitly starts an enrollment recording flow.
- Enrollment quality reports must be generated from local/in-memory takes or private profile storage, include aggregate duration/VAD/clipping/SNR/pace/alignment fields, omit raw audio and transcript text, and keep manual acceptance available because low base-model confidence alone must not reject valid accents.
- Durable accepted enrollment takes belong in `@speech/profile-manager` behind a worker-mediated profile-store client, using OPFS when available, checksummed WAV/JSON artifacts, safe path segments, reload resume, and explicit delete-all voice-data controls.
- No-gradient speaker embeddings belong in `@speech/personalization`; normalize vectors, aggregate with quality-aware weights, reject clear outliers, record channel statistics, and label deterministic signal-statistics encoders as baselines rather than trained ASR speaker models.
- Held-out base-vs-profile reports belong in `@speech/personalization`; compare aggregate-only base/profile WER, CER, code-switch, custom-term, latency, and RTF metrics by overall/language/voice-condition slices, expose a regression gate, and provide `SpeechProfileManifestV1.evaluation` metric snapshots without storing prompt text, audio, raw embeddings, or model weights.
- Profile enable/rollback/import/export/delete belongs behind `@speech/profile-manager` and the worker-mediated profile-store client; enabling must be local, applied only between utterances, base-model-compatible when an active model identity is supplied, and reversible through an active/previous profile pointer.
- Personal adapter profile manifests belong in `@speech/protocol` as `SpeechProfileManifestV1`/`SpeechProfileManifestV2`; residual adapters must bind a checksummed adapter graph file, graph-contract hash, insertion-point IDs, parameter budget, precision, utterance-boundary swap policy, and local trainer provenance without embedding raw audio, transcripts, or base model weights.
- Python trainer input validation belongs in `tools/profile-trainer/speech_profile_trainer`; validate exported profile packages before training by checking safe paths, base64 sizes, SHA-256 checksums, embedded metadata consistency, base-model manifest identity/graph-contract hash, and split enrollment datasets by prompt ID so repeated takes cannot cross train/validation/test.
- Frozen-base adapter trainer outputs must keep encoder/predictor/joiner base weights immutable, train only approved residual-adapter/speaker-conditioning groups, and emit reproducibility metadata with config/profile/base-model hashes, random seed, software versions, prompt-level splits, adapter checksum/size, and privacy flags while omitting raw audio and transcript text.
- Adapter activation gates must combine personal holdout improvement, licensed anchor-regression budget, adapter-size and RTF-overhead checks, expose `automaticActivationAllowed`, and keep reports aggregate-only with hashed or omitted prompt/case identifiers.
- Browser-compatible adapter packages must contain a `SpeechProfileManifestV1`/`SpeechProfileManifestV2` residual-adapter manifest, checksummed adapter artifact, training provenance, aggregate activation-gate evaluation, and embedded-file consistency checks; refuse packaging when the gate fails or adapter bytes do not match training metadata.
- Local trainer Docker images should copy only trainer code, required local manifest-validator code, configs, and license into the image, run the CLI as a non-root user, document `--network none` runtime commands, and keep exported profiles/adapters/evaluation inputs mounted explicitly rather than baked into layers.
- Browser adapter runtime must load the adapter as a separate removable ONNX graph, benchmark aggregate median/p95 run time and RTF overhead locally, expose adapter size/checksum metadata, and keep profile swaps limited to utterance boundaries.
- `SpeechProfileManifestV2` extends V1 with the `browser-top-adapter` adaptation member (`algorithmId=browser-top-adapter-frame-ce-v1`, `source=browser`, checksummed weights file ref, optional speaker embedding/vocabulary revision, training job/evaluation IDs). V1 manifests stay loadable through the dispatch validator; `browser-top-adapter` is rejected under V1. `migrateSpeechProfileManifestV1ToV2` is copy-on-write and metadata-only: it preserves the original V1 adaptation (CLI residual adapters are never rewritten or retrained) and bumps `schemaVersion` to 2. Storage migration must write `profile.v2.json` atomically and retain the original V1 archive until the V2 manifest and all hashes validate; a failed or interrupted migration must leave a valid V1 (or V2) profile, never a hybrid.
- Portable per-user `.speechmodel` bundles belong in `@speech/portable-model` as `PortableSpeechModelManifestV1` plus a binary envelope (`WLSPEECHMODEL\0\x01` magic, `application/vnd.wilsonle.speech.personal-model` MIME, format version 1, encrypted/unencrypted mode). The manifest binds an exact shared base model (`ExactBaseModelIdentityV1`) and a `browser-top-adapter`/`cli-residual-adapter` adaptation with checksummed weight file refs; default bundles exclude raw audio, prepared features, optimizer/checkpoint state, and the base model, and declare `containsVoiceDerivedWeights: true`. The envelope enforces a 10 MB hard ceiling, file-count/per-file/expanded-size/compression-ratio/path-segment limits, and `isSafePortableModelPath` hostile-path rejection before any allocation or decryption; encryption uses AES-256-GCM with PBKDF2-HMAC-SHA-256 (>=600,000 iterations) and never stores the passphrase.
- Custom-vocabulary enrollment prompts belong in `@speech/enrollment` as deterministic templates/scheduler output. Select enabled entries by prompt priority, preserve canonical `displayForm`/diacritics, place terms in distinct beginning/middle/end contexts, span multiple voice conditions for high-priority terms, and mark generated prompts as requiring user review before recording.
- Enrollment sentence-bank releases must be UI-independent, schema-versioned, NFC-normalized, license-linked per sentence, human-reviewed, redistributable, and include deterministic held-out IDs. Draft tooling may bypass release gates explicitly, but committed release fixtures must not contain private transcripts, corpus excerpts, recordings, or user enrollment data.
- Sentence-bank coverage and selector tooling may start with deterministic heuristic G2P, but must label it as coverage accounting rather than proof of phonetic balance until generated reports and human review support that claim. Weighted set-cover selection should maximize uncovered coverage per estimated second, penalize near-duplicate/repeated prompt templates, and keep held-out prompts excluded by default.
- Browser-only training is experimental and must never block transcription or corrupt the active profile.
- ADR 0003 records that the pinned ORT Web npm package does not expose the documented training WASM artifact or public JS training API. Production browser adaptation must therefore use a repository-owned `BrowserTrainingBackend` with fixed residual-adapter math unless a future ORT Training artifact/API is explicitly proven; it must train only frozen-feature tiny adapters through a dedicated training worker with pause/cancel/checkpoint/reload-recovery controls and the same import/checksum/regression gates as CLI adapters.
- The fixed browser adapter reference math lives in `@speech/personalization` as `residual-bottleneck-lhuc-v1`: parameter tensors are `w_down[inputDimension, rank]`, `b_down[rank]`, `w_up[rank, inputDimension]`, `b_up[inputDimension]`, and `lhuc[inputDimension]`; forward output is `(input + residualScale * (tanh(input·w_down + b_down)·w_up + b_up)) * (2 * sigmoid(lhuc))`; all-zero tensors preserve identity because the residual is zero and LHUC logit zero maps to unit scale. Keep parameter/byte budget checks and tensor-contract validation with this reference module.
- The browser-training prototype in `@speech/personalization` is a small deterministic affine/residual-adapter trainer over frozen features; keep it UI-independent, parameter-budgeted, local-only, and activation-gate-required, and do not route it through the real-time ASR worker.
- Browser-training checkpoint, pause, cancel, and reload-recovery flows must leave the previous active profile intact. Checkpoints are resumable training state only, not activation artifacts; activation still requires import/checksum/compatibility/regression gates.
- Browser-training readiness policy and coverage reporting belongs in `@speech/enrollment` with profile-store mapping in `@speech/profile-manager`: report aggregate accepted-take counts/duration by language, voice condition, redacted prompt identity, and redacted vocabulary identity, set `automaticTrainingAllowed` only when every policy requirement passes, and never expose raw prompt IDs, transcript text, vocabulary terms, audio, feature tensors, checkpoints, or adapter weights.
- Browser training jobs must freeze accepted enrollment and active vocabulary revisions through `@speech/profile-manager`/the profile-store worker before feature preparation starts; store immutable local revision JSON outside default profile exports, bind utterance metadata/audio refs and vocabulary revision content by SHA-256, return only redacted worker summaries/verification results to UI code, and fail verification when later profile or vocabulary edits change the frozen inputs.
- Prompt-identity train/validation/test splitting for browser training belongs in `@speech/enrollment` with profile-store mapping in `@speech/profile-manager`: use seeded deterministic stratification over frozen prompt IDs, languages, voice conditions, and durations; keep all repeated takes and voice-condition variants for a prompt in the same split; expose only redacted aggregate split summaries outside worker/profile-owned code.
- Browser-vs-Python adapter comparison belongs in `@speech/personalization` as an aggregate-only report that checks base-model compatibility, browser/Python activation-gate status, quality parity, RTF/training-duration overhead, adapter size, and explicitly reports insufficient evidence when browser held-out quality metrics are missing.
- Browser-only adapter training was deferred for the current shipped baseline by `docs/adr/0001-defer-browser-training-path.md`; production v0.5.0 Personal Voice Models must follow `docs/adr/0002-v0-5-0-personal-model-semantics.md` and keep the browser path gated until feasibility, quality, privacy, security, portability, and release validation pass.
