# Testing and validation

Run the smallest useful validation set for each change, but keep these repository-level commands healthy:

```bash
pnpm lint
pnpm format-check
pnpm typecheck
pnpm test
pnpm build
pnpm chromium-smoke
```

Required CI checks are lint, format-check, typecheck, unit tests, Python tests, production build, Chromium smoke, and license/generated-file checks.

Performance gates must be measured from audio timestamps and documented with browser, OS, hardware, model version, provider, thread count, and power state. Shared CI performance is informational unless it runs on designated reference hardware.

- Numerical parity fixtures (log-Mel, transcript, tiny-adapter) live under `test-data/expected/*.json` and are regenerated by `tools/*/generate_*_reference.py`; TS tests load the fixture and compare against the in-package implementation, while Python tests reproduce the fixture from the reference module. The tiny-adapter fixture (`test-data/expected/tiny-adapter-parity.json`) pins a deterministic 50-epoch frozen-feature affine-adapter trajectory (initial/final loss, per-checkpoint weights/bias/checksum, and runtime-adapter forward output) shared by `@speech/personalization` browser math and `tools/training-parity`; when changing adapter math, regenerate with `PYTHONPATH=tools/training-parity uv run python tools/training-parity/generate_tiny_adapter_reference.py` and confirm both the TS and Python parity tests still pass within tolerance.
