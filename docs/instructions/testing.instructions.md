# Testing and validation

Run the smallest useful validation set for each change, but keep these repository-level commands healthy:

```bash
pnpm lint
pnpm format-check
pnpm typecheck
pnpm test
pnpm build
pnpm chromium-smoke
```

Required CI checks are lint, format-check, typecheck, unit tests, Python tests, production build, Chromium smoke, and license/generated-file checks.

Performance gates must be measured from audio timestamps and documented with browser, OS, hardware, model version, provider, thread count, and power state. Shared CI performance is informational unless it runs on designated reference hardware.

- Numerical parity fixtures (log-Mel, transcript, tiny-adapter) live under `test-data/expected/*.json` and are regenerated by `tools/*/generate_*_reference.py`; TS tests load the fixture and compare against the in-package implementation, while Python tests reproduce the fixture from the reference module. The tiny-adapter fixture (`test-data/expected/tiny-adapter-parity.json`) pins a deterministic 50-epoch frozen-feature affine-adapter trajectory (initial/final loss, per-checkpoint weights/bias/checksum, and runtime-adapter forward output) shared by `@speech/browser-training` backend math and `tools/training-parity`; when changing adapter math, regenerate with `PYTHONPATH=tools/training-parity uv run python tools/training-parity/generate_tiny_adapter_reference.py` and confirm both the TS and Python parity tests still pass within tolerance. If browser-training defaults move to AdamW/scheduled optimization, keep the parity fixture on its explicit `sgd` trajectory and add separate focused tests for sampler, AdamW, validation, early-stopping, gradient-clipping, finite-value guards, resume-state checksum validation, pause/resume output matching uninterrupted training, Web Locks/BroadcastChannel cross-tab coordination redaction/busy-state behavior, ASR-priority cooperative pause, and session resource cleanup/disposal.
- Personal Models navigation/profile-card/preflight UI changes need focused helper tests for aggregate-only card/preflight data, Playwright coverage for empty/generic-fallback, capability, training-companion, missing-recording-task, saved-profile, and import-mode states, and a browser visual review across desktop/mobile when layout changes. Multi-profile lifecycle changes also need package tests for list/rename/dedupe/replace/import-as-new/name-collision/delete-active-pointer behavior and worker/client typecheck coverage. Avoid duplicating exact public model display names across unrelated sections when existing strict locators depend on unique text.
- Browser-training progress/recovery UI changes need focused helper tests for named phase mapping, redacted recovery/resource summaries, safe-boundary pause/cancel/restart/resume controls, live-region progress text, text equivalents for phase/progress visuals, destructive-action confirmations, reload recovery after page refresh, cross-tab lock contention, ASR-priority pause, reduced-motion/narrow-viewport accessibility checks, and desktop/mobile visual review when layout changes. Cross-browser/fault-injection hardening should keep Chromium green by default and make Edge opt-in (`SPEECH_E2E_EDGE=1`) on hosts with Microsoft Edge installed; cover worker faults, quota/recovery failure, cross-tab contention, hostile imports, export/import, activation, rollback, and deletion without adding private fixtures.
- Personal/anchor end-to-end evaluation changes need focused package tests for `generic`/`p1`/`candidate` aggregate metrics, personal-heldout improvement, anchor and slice regression gates, RTF overhead relative to P1, adapter-size limits, false-insertion budgets, and privacy serialization checks that reject raw case IDs, text, selected-vocabulary IDs, audio, feature tensors, checkpoints, or adapter weights. Bilingual cohort gate changes need tests for missing-data release blocking, 30-speaker pass/fail math, language/accent/voice-condition slice regression, duplicate speaker input rejection, and privacy serialization that omits raw speaker/profile identifiers. Activation-decision and comparison UI changes also need hard-vs-soft gate tests, advanced override acceptance/rejection tests, rollback/generic-fallback assertions, worker-client guarded-enable tests, and desktop/mobile visual review when rendered UI changes.
- Portable `.speechmodel` encryption/importer changes need focused tests for unencrypted explicit-export round trip, encrypted AES-GCM/PBKDF2 round trip, deterministic salt/IV fixture behavior, no plaintext leakage in envelope bytes, minimum iteration enforcement, missing/wrong passphrase handling, tampered ciphertext rejection, abort checks, controlled Web Crypto/base64 error paths, envelope-to-inner-bundle importer round trips, wrong-passphrase importer rejection, hostile payload rejection for raw privacy flags, base-model/operator/external-data media/types, unsafe paths, duplicate/colliding files, missing refs, checksum mismatch, trailing bytes, malformed JSON, archive-bomb file counts/arrays, forbidden tensor payload keys, parser fuzz mutations for invalid magic/version/GCM tag, plus exact-base compatibility, worker-owned smoke-vector execution, runtime-smoke timeout cleanup, temporary staging cleanup on failure, committed-record-last atomic import behavior, and CLI residual-adapter `.speechmodel` coverage that preserves the original residual-adapter profile manifest, requires exact base/tokenizer identity, and proves unencrypted CLI output is explicit opt-in. Speech profile V1-to-V2 migration changes need a committed synthetic v0.4.0 fixture plus package tests for V1 dispatch loading, copy-on-write `profile.v2.json`, stale temp cleanup, valid-target resume, invalid-target replacement from retained V1, missing-source failure, V2 residual-adapter runtime compatibility, and aggregate-only migration summaries.
